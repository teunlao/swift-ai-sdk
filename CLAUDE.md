## Roles & Workflow

### Executor Role (Codex Agent)
**Implement features, write tests, create validation request.**

Executor agent workflow:
1. Receive task via prompt from orchestrator
2. Find TypeScript in `external/vercel-ai-sdk/packages/`
3. Port to appropriate Swift package
4. Port ALL upstream tests
5. Run `swift build && swift test` (must pass 100%)
6. Create validation request in `.validation/requests/validate-TASK-YYYY-MM-DD.md`
7. **Stop and wait** - orchestrator will handle validation workflow

**üö® CRITICAL Rules for Executor Agents**:
- ‚ùå **NEVER TOUCH OTHER AGENTS' WORK** ‚Äî Only edit files in your task scope
- ‚úÖ Create `.validation/requests/*.md` file and stop
- ‚ùå Do NOT call MCP tools (agents can't - only orchestrator can)
- ‚ùå Do NOT launch validator (orchestrator does this)

### Validator Role (Codex Agent)
**Review implementation, compare with upstream, generate report.**

Validator agent workflow:
1. Receive validation request path via prompt
2. Read `.validation/requests/*.md` file
3. Examine implementation files
4. Compare with upstream TypeScript (line-by-line)
5. Verify ALL tests ported
6. Create `.validation/reports/report-*.md` with verdict (APPROVED/REJECTED)
7. **Stop and wait** - orchestrator will handle status updates

**üö® CRITICAL Rules for Validator Agents**:
- ‚úÖ Work in executor's worktree (same directory)
- ‚úÖ Check EVERY requirement from validation scope
- ‚úÖ Be thorough - reject if ANY issue found
- ‚ùå Do NOT call MCP tools (agents can't - only orchestrator can)

### Orchestrator Workflow (Your Role)
**üö® YOU MUST ACTIVELY MONITOR AND EXECUTE THIS WORKFLOW - NOT PASSIVE!**

**NON-NEGOTIABLE VALIDATION RULE:** Execute the full validation workflow for 100% of tasks‚Äîevery single one. The ONLY time you may skip validation is when the user explicitly says exactly `skip validation` or `no validation needed`. Any other wording still requires full validation, regardless of urgency or perceived scope.

**STEP 1: Launch Executor**
```
launch_agent(role='executor', worktree='auto', prompt='...')
```
‚Üí Returns `agent_id` (e.g., executor-1760461163346)
‚Üí Agent starts working

**STEP 2: MONITOR Executor Until Completion**

**STEP 3: IMMEDIATELY Call request_validation() - DON'T WAIT FOR USER!**
```
request_validation(executor_id=executor_id, summary='...')
```
‚Üí Returns `validation_id`
‚Üí Executor status ‚Üí 'blocked'
‚Üí **DO THIS AS SOON AS EXECUTOR FINISHES, NOT WHEN USER ASKS!**

**STEP 4: Launch Validator in Executor's Worktree**
```
# Get executor's worktree from status() or validation session
launch_agent(
  role='validator',
  worktree='manual',
  cwd='/path/to/executor-worktree',
  prompt='Validate executor work...'
)
```
‚Üí Returns `validator_id`

**STEP 5: Assign Validator to Validation Session**
```
assign_validator(validation_id=validation_id, validator_id=validator_id)
```
‚Üí Links validator to session
‚Üí Session status ‚Üí 'in_progress'

**STEP 6: MONITOR Validator Until Completion**
‚Üí Periodically check status() and get_logs() until validator completes
‚Üí When `idle_minutes > 1` and validation report created, proceed to STEP 7

**STEP 7: IMMEDIATELY Call submit_validation() - DON'T WAIT FOR USER!**
```
submit_validation(
  validation_id=validation_id,
  result='approved' or 'rejected',
  report_path='.validation/reports/...'
)
```
‚Üí If approved: executor status ‚Üí 'validated', ready to merge
‚Üí If rejected: executor status ‚Üí 'needs_fix', must fix bugs
‚Üí **DO THIS AS SOON AS VALIDATOR FINISHES, NOT WHEN USER ASKS!**

**STEP 8: If Approved - Merge; If Rejected - Continue Executor**

**If APPROVED:**
```bash
# 1. Commit changes in executor's worktree
cd /path/to/executor-worktree
git add .
git commit -m "feat: descriptive message

ü§ñ Generated with [Claude Code](https://claude.com/claude-code)

Co-Authored-By: Claude <noreply@anthropic.com>"

# 2. Merge executor branch to main
cd /main/repo/path
git merge agent/executor-TIMESTAMP --no-edit

# 3. Cleanup worktree
kill_agent(executor_id, cleanup_worktree=true)
```

**If REJECTED:**
```
continue_agent(executor_id, 'Fix bugs from validation report at .validation/reports/...')
# Then GO BACK TO STEP 2 and repeat cycle!
```

**üö® CRITICAL RULES:**
1. **YOU ACTIVELY DRIVE THE WORKFLOW** - don't wait for user commands after each step
2. **MONITOR agents continuously** - check status/logs every 30-180s until completion
3. **IMMEDIATELY call MCP tools** when agent finishes (request_validation, submit_validation)
4. **DON'T ASK USER "what next?"** - you know the workflow, execute it!
5. **Validation is iterative** - rejected ‚Üí fix ‚Üí re-validate ‚Üí repeat until approved
6. **Validation never skipped** - follow the mandate above unless the user literally says `skip validation` or `no validation needed`

**‚è±Ô∏è SLEEP BEFORE MONITORING (MANDATORY):**
- **ALWAYS use sleep && before monitoring commands**
- Write as ONE Bash command: `sleep N && monitoring_command`
- Start with 30s, increase to 60s for complex tasks, up to 180s for very long tasks
- Examples:
  ```bash
  Bash: sleep 30 && status(agent_id="executor-123")
  Bash: sleep 60 && get_logs(agent_id="executor-123")
  Bash: sleep 120 && status(agent_id="executor-123")
  ```

**‚ö†Ô∏è WORKTREE MODE - MANDATORY DEFAULTS:**

**ALWAYS use these modes unless user EXPLICITLY requests otherwise:**

- **Executors:** `worktree="auto"` (creates isolated Git worktree + unique branch)
  - ‚úÖ Use for: ALL executor tasks (Swift porting, bug fixes, features, tooling, docs, ANY code changes)
  - ‚úÖ Isolation prevents conflicts between parallel agents
  - ‚ùå NEVER use `manual` unless user explicitly says "use manual mode"

- **Validators:** `worktree="manual"` + `cwd=executor_worktree` (works in executor's directory)
  - ‚úÖ Use for: ALL validator tasks (must access executor's files)
  - ‚úÖ Validator needs same directory to review executor's work
  - ‚ùå NEVER use `auto` for validators (creates separate worktree, can't access executor files)

**Examples of user requests that DO NOT override defaults:**
- "fix the CSS" ‚Üí executor with `worktree="auto"` ‚úÖ
- "update the dashboard" ‚Üí executor with `worktree="auto"` ‚úÖ
- "make this change" ‚Üí executor with `worktree="auto"` ‚úÖ
- "–ø–æ–ø—Ä–æ—Å–∏ –∞–≥–µ–Ω—Ç–∞ —Å–¥–µ–ª–∞—Ç—å X" ‚Üí executor with `worktree="auto"` ‚úÖ

**Only override if user says:**
- "use manual mode" ‚Üí executor with `worktree="manual"` ‚úÖ
- "work in main directory" ‚Üí executor with `worktree="manual"` ‚úÖ
- "don't create worktree" ‚Üí executor with `worktree="manual"` ‚úÖ

## Parity Standards

### Must Match
- ‚úÖ Public API (names, parameters, types)
- ‚úÖ Behavior (edge cases, errors)
- ‚úÖ Error messages (same text)
- ‚úÖ Test scenarios (all ported)

### Allowed Adaptations
- ‚úÖ `Promise<T>` ‚Üí `async throws -> T`
- ‚úÖ `AbortSignal` ‚Üí `@Sendable () -> Bool`
- ‚úÖ Union types ‚Üí `enum` with associated values
- ‚úÖ `undefined` ‚Üí `nil`
- ‚úÖ `Record<K, V>` ‚Üí `[K: V]`

**Document adaptations** with rationale. See `plan/principles.md`.

---

## TypeScript ‚Üí Swift Patterns

| TypeScript | Swift |
|------------|-------|
| `Promise<T>` | `async throws -> T` |
| `value?: T \| undefined` | `value: T? = nil` |
| `type A \| B` | `enum Result { case a(A), case b(B) }` |
| `Record<K, V>` | `[K: V]` |
| `AbortSignal` | `@Sendable () -> Bool` |

## Key Principles

1. **üö® YOU orchestrate everything** ‚Äî Agents create files, YOU call MCP commands for workflow
2. **Agents can't call MCP tools** ‚Äî Only YOU can use orchestrator commands
3. **Validation is 3-step process** ‚Äî request_validation ‚Üí assign_validator ‚Üí submit_validation
4. **Executor creates request file** ‚Äî Then YOU call request_validation(executor_id)

7. **Never commit without permission** ‚Äî Explicit user request required
8. **Worktree isolation** ‚Äî Each executor gets own directory, validator shares it
